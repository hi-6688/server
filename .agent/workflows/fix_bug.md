---
description: 修復 Bug 的標準流程，確保定位準確並避免引入新問題
---
請完全依照以下步驟執行：

### 第一階段：確認問題
1. **蒐集資訊**：詢問使用者 Bug 的具體表現（錯誤訊息、復現步驟、影響範圍）。
2. **閱讀相關程式碼**：定位可能出問題的檔案，先完整閱讀理解邏輯。
3. **閱讀相關文件**：確認 `docs/` 對該功能的描述，理解預期行為。

### 第二階段：定位與修復
4. **建立修復分支**：
// turbo
```bash
git checkout -b fix/<bug描述>
```
5. **定位根因**：找出 Bug 的根本原因，跟使用者說明。
6. **最小化修改**：只修必要的部分，不要順手重構其他東西。
7. **驗證修復**：用指令或瀏覽器確認 Bug 已修復。

### 第三階段：收尾
8. **確認無副作用**：檢查修改是否影響其他功能。
9. **Commit**：
// turbo
```bash
git add -A && git commit -m "fix(<範圍>): <修了什麼問題>"
```
10. **合併回 dev**：
```bash
git checkout dev && git merge fix/<bug描述>
```
