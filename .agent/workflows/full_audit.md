執行全面盤點時，請恪守以下目標：**發現不一致 → 立即修正檔案 → 產出報告**。

### 第一階段：全域掃描
掃描 `servers/` 根目錄與主要子目錄，獲取當前真實的檔案結構。

### 第二階段：核心文件審查
逐一讀取下列文件，並進行程式碼交叉比對：

**1. 專案進度與結構：**
- [ ] `README.md`：目錄樹是否正確？Dev Log 是否有遺漏？
- [ ] `ROADMAP.md`：任務狀態是否需推進？

**2. 架構藍圖 (`docs/`)：**
- [ ] `FRONTEND_ARCH.md`：前端元件與實體程式碼是否一致？
- [ ] `BACKEND_API.md`：API 路由與 `web_interface` 是否一致？
- [ ] `VM_ARCHITECTURE.md`：雙機通訊邏輯是否與代理腳本相符？
- [ ] `HiHi_Proposal.md`：AI 模型版本與記憶體架構（含雙層哲學設定）是否與最新程式碼一致？

**3. 規則與自動化：**
- [ ] `.agent/rules/readrules.md`：索引是否涵蓋所有的 `docs/` 文件？
- [ ] `.agent/workflows/*.md`：邏輯是否皆有效？

### 第三階段：修正與收尾
1. **立即自動修正**：若發現任何資訊過期，**嚴禁只給建議**。你必須直接呼叫編輯工具（例如 `replace_file_content` 或 `multi_replace_file_content`）直接把過期的文件內容更新為最新的正確資訊。
2. **產出 Artifact**：建立 `audit_report.md`，以表格條列所有被你自動修正的項目與內容差異。
3. **清理廢棄物**：在報告中列出已棄用但未刪除的檔案，供使用者確認。
