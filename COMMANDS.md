# 🤖 神奇嗨螺指令手冊 (Command Manual)

此文件列出機器人目前支援的所有指令以及其使用限制。

## 📍 頻道使用規則

| 頻道類型 | 允許的指令類別 | 備註 |
| :--- | :--- | :--- |
| **泰拉瑞亞頻道** (一般) | 僅限 **查詢** (`status`) | 禁止使用任何管理指令 |
| **測試/日誌頻道** (管理) | **所有指令** (`start`, `stop`, `cmd`...) | 這是管理員專用的後台 |

---

## 🛠️ 指令列表

### 🟢 一般查詢 (所有人可用)
| 指令 | 說明 | 參數 | 權限限制 |
| :--- | :--- | :--- | :--- |
| `/status` | 查看伺服器狀態、IP、線上人數 | 無 | 🌍 全域可用 |
| `/help` | 顯示說明 (Discord 內建) | 無 | 🌍 全域可用 |

### 🔒 管理員指令 (僅限日誌頻道)
> **注意**：在一般頻道使用這些指令會被拒絕。

| 指令 | 說明 | 參數 | 風險等級 |
| :--- | :--- | :--- | :--- |
| `/start` | 喚醒 Terraria 伺服器 | 無 | 🟢 低 (防呆：已開啟不會重複開) |
| `/stop` | **[危險]** 強制關閉伺服器 | 無 | 🔴 高 (會先執行 save 再關閉) |
| `/ping` | 測試機器人延遲 | 無 | 🟢 低 |
| `/cmd` | 發送後台 RCON 指令 | `command_text`: 例如 `kick Player` | 🟠 中 (操作不當可能影響遊戲) |
| `/reload` | **[開發]** 熱重載 Python 模組 | 無 | 🟠 中 (僅更新程式碼，不重啟進程) |

---

## ❓ 常見問題
**Q: 為什麼我在一般頻道看不到 `/start` 指令？**
A: 請檢查您的 Discord 伺服器設定 (`整合` -> `機器人`), 可能已被管理員從 UI 層級隱藏。

**Q: 為什麼我在一般頻道輸入 `/start` 出現「權限不足」？**
A: 這是程式碼層級的保護 (Hard Limit)，即使 UI 顯示了，機器人也會拒絕執行。

## 💻 系統管理員指令 (VM1 Docker 操作)
以下指令請在 VM1 的 `/home/terraria/servers/` 終端機內執行：

| 動作 | 指令 | 說明 |
| :--- | :--- | :--- |
| **一鍵啟動** | `docker compose up --build -d` | 構建並在背景啟動所有服務 (Web, HiHi, Conch) |
| **關閉所有** | `docker compose down` | 停止並移除所有容器 |
| **查看日誌** | `docker compose logs -f` | 查看所有容器的即時日誌 (`-f` bot-hihi 則只看嗨嗨) |
| **熱重載 Web** | `docker compose restart web-api` | 修改 `api.py` 後，執行此指令讓新代碼生效 |
| **熱重載 Bot** | `docker compose restart bot-hihi` | 修改 AI 邏輯後，重啟靈魂容器 |
